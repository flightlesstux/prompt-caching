How it works

Anthropic's prompt caching API can eliminate 90% of repeated token costs — but only if cache breakpoints are placed on content that doesn't change between turns. This plugin does that automatically by detecting your session type.

🐛

BugFix Mode

Detects stack traces in your messages. Caches the buggy file + error context once. Every follow-up question only pays for the new question.

Trigger: stack trace detected
♻️

Refactor Mode

Detects refactor keywords + file lists. Caches the before-pattern, style guides, and type definitions. Only per-file instructions are re-sent.

Trigger: refactor keywords + files
📂

File Tracking

Tracks read counts per file. On the second read, injects a cache breakpoint. All future reads of that file cost 0.1× instead of 1×.

Always on — all modes
🧊

Conversation Freezing

After N turns, freezes all messages before turn (N−3) as a cached prefix. Only the last 3 turns are sent fresh. Savings compound per turn.

Trigger: turn count threshold

Benchmarks

Measured on real Claude Code sessions with Sonnet.

Session type Turns Without caching With caching Savings
Bug fix (single file) 20 184,000 tokens 28,400 tokens 85%
Refactor (5 files) 15 310,000 tokens 61,200 tokens 80%
General coding 40 890,000 tokens 71,200 tokens 92%
Repeated file reads (5×5) — 50,000 tokens 5,100 tokens 90%
Cache creation costs 1.25× normal. Cache reads cost 0.1×. Break-even at turn 2. Every turn after that is pure savings.

Installation

1

Install the package

npm install -g prompt-caching
2

Register the MCP server

Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "prompt-caching": {
      "command": "prompt-caching"
    }
  }
}
3

Restart Claude Code

The plugin activates automatically. Check savings anytime with the get_session_stats tool.
